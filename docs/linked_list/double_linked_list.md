# Project programming double_linked_list in C
    C 언어를 사용하여 이중 연결 리스트 구현

### 설계 명세
    구현에 필요한 조건 서술

### 제한사항
    **Type** `int` (정수형) : 가장 기본적인 데이터 자료형으로 구현했습니다. 
    **중복없음** : 중복을 처리하는 경우는 상정하지 않았습니다. 
    **Dummy Node** 사용 : 로직 Simplify

### 자료구조
    **이중연결리스트**
    *이중연결리스트를 구현할 때, **단일연결리스트**와의 차이점에 주목하여 구현했습니다. 

### 단일연결리스트 VS 이중연결리스트
    **Node구조체** : Prev 포인터 추가
    **List구조체** : Tail 노드 포인터 추가
    **메서드**
    *삽입/삭제/조회(탐색) 시간 복잡도
    **삽입** : O(1) *단일연결리스트와의 가장 큰 차이
    **삭제** : O(N) 동일하다. 
    **조회** : O(N) 동일하다. 

### 개선 가능성
    **삭제/조회**에서 개선점
    *이중연결리스트의 Prev를 적절히 사용하지 못하고 있다. 
    *List 구조체에 size 저장 변수 추가. 
    *삭제/조회할 데이터의 인덱스를 따로 관리한다면, Head / Tail 시작점 고르기 용이. 하지만 큰 차이는 없을 듯 하다. 

### 한계
    **시작점**이 강제된다. **배열**의 장점은 **인덱스**를 가지고 있다는 점이다. 
    반대로 **리스트**의 장점은 크기를 미리 지정하지 않아도 된다는 점.(메모리 공간 효율)

### 결론
    **리스트**의 한계인 삭제와 조회 -> **인덱스**가 필요한 경우는 Array 사용 : HashMap, 이진탐색트리(BST/우선순위 큐), 인접행렬
    **리스트**의 장점인 메모리 공간 효율, 크기 미지정이 필요한 경우 List 사용 : Stack, Queue, Graph(인접리스트)

    **스택과 큐**는 삭제와 삽입을 항상 첫/끝 노드에서 작업하므로 리스트의 단점이 없어진다.
