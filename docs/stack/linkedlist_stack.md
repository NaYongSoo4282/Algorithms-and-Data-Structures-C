## 1. Project Goal
### 구현한 자료구조(연결리스트, 배열)를 가지고 자료구조 ADT를 구현한다. 
### 스택 ADT를 구현한다. 

## 2. Challenge Point
### 매번 구현하지 않고, 구현해둔 코드를 재사용하는 것 
### 스택 자료구조의 특징 **FILO**을 구현하는 것

## 3. Array Vs LinkedList
### [배열의 한계] : 크기 지정의 한계, 삽입/삭제 시 배열 요소의 재배치 [배열의 장점] : 인덱스를 통한 접근 비용, 빠른 생성 속도
### [연결리스트의 한계] : 중간 노드 접근 비용 [연결리스트의 장점] : 크기의 탄력성, 삽입/삭제
### [결론] : 최대 인덱스를 알 수 있다면 배열, 모른다면 연결리스트가 더 효율적일 것이다

## 구현 상세
### 1. 저장 데이터 -> int 
### 2. 기본 메서드 -> 삽입/삭제/접근 등 이중연결리스트를 가져와서 사용
